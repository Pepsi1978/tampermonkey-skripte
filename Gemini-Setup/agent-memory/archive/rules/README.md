# Gemini Regel-Archiv
Dies ist das Langzeitgedaechtnis fuer alle Regeln, die geaendert, ersetzt oder entfernt wurden.

## Warum dieses Archiv existiert
- **Meta-Intelligenz:** Wir dokumentieren, *warum* sich Regeln weiterentwickeln.
- **Fehlerschutz:** Verhindert das Reaktivieren von Mustern, die sich als unbrauchbar oder unerwuenscht erwiesen haben.
- **Kontinuitaet:** Neue Agenten koennen die Evolutionsgeschichte des Setups nachvollziehen.

## Archivierungs-Prozess
1. Wenn eine Regel signifikant geaendert oder entfernt wird, erstelle ein neues MD-File in `Gemini-Setup/agent-memory/archive/rules/`.
2. Benenne es nach dem Schema `YYYY-MM-DD-kurzbeschreibung.md`.
3. Dokumentiere Datum, Grund und den alten Stand.
